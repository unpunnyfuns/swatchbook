---
title: Sample blocks
---

Lower-level than the per-type overview blocks: these render the visual for **one** already-realised token. Useful for custom MDX layouts that don't want the full overview chrome, and for embedding a single sample next to prose. Each is a built-in [presenter](/reference/blocks/presenters), so it takes the uniform `PresenterProps` shape (`path`, `token`, `cssVar?`, `colorFormat`, `options?`) rather than a bare path: a presenter renders a token it's handed, it doesn't look one up.

## MotionSample

```tsx
<MotionSample path={path} token={token} colorFormat={colorFormat} options={{ speed: 1 }} />
```

Animated ball + track for one `transition` / `duration` / `cubicBezier`. `options.speed?: MotionSpeed` (default `1`); `options.runKey?: number` forces a restart when it changes.

```ts
import type { MotionSpeed } from '@unpunnyfuns/swatchbook-blocks';
type MotionSpeed = 0.25 | 0.5 | 1 | 2;
```

The four allowed playback rates the speed pills surface above the sample track. Re-exported for hosts that want to drive the speed externally.

## ShadowSample

```tsx
<ShadowSample path={path} token={token} colorFormat={colorFormat} />
```

Surface rectangle with the token's shadow applied as `box-shadow`.

## BorderSample

```tsx
<BorderSample path={path} token={token} colorFormat={colorFormat} />
```

Surface rectangle with the token's border applied.

## DimensionSample

```tsx
<DimensionSample path={path} token={token} colorFormat={colorFormat} options={{ visual: 'length' }} />
```

Width-driven bar, border-radius square, or sized square for one `dimension` token. `options.visual?: 'length' | 'radius' | 'size'` (default `'length'`).
